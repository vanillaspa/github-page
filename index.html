01 — Installation

Quick Start

Vanilla SPA is built for AI-ready development of W3C standards-conforming web-components. No magic, no virtual DOM — just the platform.

Grab the boilerplate

git clone https://github.com/vanillaspa/boilerplate.git

Or set up manually:

1

Create your project

npm init -y
npm install @vanillaspa/web-components @vanillaspa/event-bus @vanillaspa/sqlite-database
npm install vite --save-dev
2

Configure scripts

{
  "type": "module",
  "scripts": {
    "dev":     "vite",
    "build":   "vite build",
    "preview": "vite preview"
  }
}

For SQLite WASM, you MUST configure Vite.

3

Set up the folder structure

project/
├── index.html
├── package.json
└── src/
    ├── components/
    │   └── app/
    │        └── app-start.html
    ├── contracts/
    │   └── sqlite.contract.js
    └── main.js

@vanillaspa/web-components scans /src/components. All found components are added automagically to your customElements registry.

4

Run dev server

npm run dev

02 — Components

Your First Component

Filename convention: /src/components/{prefix}/{prefix}-{name}.html

app-start.html           →  <app-start>
app-start-page-home.html →  <app-start-page-home>

File: /src/components/app/app-start.html

<!-- The three sections must be siblings at the top level -->

<template>
  <header>
    <h1>My custom element</h1>
  </header>
  <main>
    <p>Hello World!</p>
  </main>
</template>

<style>
  :host { display: block; }

  header {
    padding: 1rem 2rem;
    background: #f9fafb;
    border-bottom: 1px solid #e5e7eb;
  }

  h1 { color: #ff69b4; font-size: 1.5rem; }
</style>

<script>
  // shadowDocument is automatically available
  const h1 = shadowDocument.querySelector('h1');
  console.log('Component loaded:', h1.textContent);
</script>

The <script> body is compiled into an AsyncFunction with shadowDocument (your shadow root) passed as a parameter. Top-level await and dynamic import() work out of the box.

Important: <template>, <style>, and <script> must be siblings at the top level — not nested inside each other.

03 — Shadow DOM

Web-Components

Vanilla SPA passes shadowDocument directly into every component script. It's your component's Shadow Root — no setup needed.

<script>
  // Direct Shadow DOM access
  const button = shadowDocument.getElementById('my-btn');
  const output = shadowDocument.querySelector('.result');

  button.addEventListener('click', () => {
    output.textContent = 'Clicked!';
  });

  // Access the host element
  const host = shadowDocument.host;
  console.log(host.tagName); // "APP-START"
</script>

Note: Scripts run as async functions, not ES modules. Use dynamic import() instead of static import statements. Use the data-action pattern for functions called from templates.

04 — Event Bus

Event Bus

@vanillaspa/event-bus provides context-scoped events. Listeners are automatically cleaned up when a component disconnects — no manual teardown needed.

Setup in src/main.js

import('@vanillaspa/web-components');

customElements.whenDefined("router-app2").then(() => {
    Promise.all([
        import('@vanillaspa/event-bus'),
        import('@vanillaspa/sqlite-database'),
        ...Object.values(
            import.meta.glob('/src/contracts/*.contract.js', { eager: true })
        ),
    ]).then((importedModules) => {
        importedModules.forEach((module) => {
            if (!module.name) throw new Error('Missing name in imported module.');
            window[module.name] = Object.freeze({ ...module });
        });
    }).finally(() => {
        document.body.replaceChildren();
        document.body.appendChild(document.createElement('router-app2'));
    });
});

Every imported module — npm packages and local contracts alike — is frozen and exposed on window under its name export. This makes window.eventbus, window.sqlite, window.SqliteContract, etc. available to all component scripts without explicit imports.

In your component

<script>
const host = shadowDocument.host;

// Listen for events scoped to this component instance
eventbus.addEventListener('user:action', (e) => {
    console.log('Action:', e.detail.action);
}, host);

// Dispatch an event scoped to this component instance
eventbus.dispatchEvent(
    new CustomEvent('user:action', {
        detail: { action: 'clicked' }
    }),
    host
);
</script>

05 — Event Contract

Event Contract

An Event Contract is a plain data file in src/contracts/ that declares the event names, detail shapes, and response patterns for a namespace. It has no runtime behaviour — it is the source of truth an AI CLI reads to generate correct addEventListener and dispatchEvent calls.

Add a *.contract.js file and it is automatically picked up by main.js and exposed on window.

src/contracts/sqlite.contract.js

export const name      = 'SqliteContract';
export const namespace = 'sqlite';
export const events    = {
    create:    { detail: { name: 'string' },                                 past: 'created'    },
    query:     { detail: { sql: 'string', name: 'string' },                  past: 'queried'    },
    statement: { detail: { sql: 'string', values: 'array', name: 'string' }, past: 'executed'   },
    close:     { detail: { name: 'string' },                                 past: 'closed'     },
    delete:    { detail: { name: 'string' },                                 past: 'deleted'    },
    upload:    { detail: { fileName: 'string', arrayBuffer: 'any' },         past: 'uploaded'   },
    download:  { detail: { name: 'string' },                                 past: 'downloaded' },
};

Naming convention

request sqlite:{action} 2-fold · scoped to host
response sqlite:{past}:{event.timeStamp} 3-fold · correlated
error sqlite:error:{event.timeStamp} 3-fold · correlated

Bridge side — the component that handles requests

<script>
const host = shadowDocument.host;

eventbus.addEventListener('sqlite:query', async (event) => {
    const { sql, name } = event.detail;
    const result = await sqlite.executeQuery(sql, name);
    eventbus.dispatchEvent(
        new CustomEvent(`sqlite:queried:${event.timeStamp}`, { detail: { result } }),
        host
    );
}, host);
</script>

Consumer side — any component that needs data

<script>
const host = shadowDocument.host;
const req  = new CustomEvent('sqlite:query', {
    detail: { sql: 'SELECT * FROM notes', name: 'notes' }
});

eventbus.addEventListener(`sqlite:queried:${req.timeStamp}`, (e) => {
    console.log(e.detail.result);
}, host);

eventbus.dispatchEvent(req, host);
</script>

event.timeStamp is the correlation key — each request gets its own response listener, so concurrent calls never interfere with each other.

06 — Database

Database

@vanillaspa/sqlite-database provides a full SQLite database in the browser, persisted in OPFS, running in a dedicated Web Worker per database.

<script>
// Create (or reopen) a named database
await sqlite.createDB('my-app');

// DDL — create a table
await sqlite.executeQuery(`
  CREATE TABLE IF NOT EXISTS users (
    id    INTEGER PRIMARY KEY,
    name  TEXT,
    email TEXT
  )
`, 'my-app');

// Parameterised insert — SQL injection safe
await sqlite.executeStatement(
    'INSERT INTO users (name, email) VALUES (?, ?)',
    ['Anna', 'anna@example.com'],
    'my-app'
);

// Query
const users = await sqlite.executeQuery(
    'SELECT * FROM users',
    'my-app'
);
</script>

07 — Templates

Using Functions in Templates

Component scripts run as async functions, so declared functions are not global. Use the data-action pattern with bindActions() to wire template buttons to script functions.

Create /src/bindActions.js

export function bindActions(shadowDocument, actions) {
    shadowDocument.querySelectorAll('[data-action]').forEach(el => {
        const fn = actions[el.dataset.action];
        if (fn) {
            eventbus.addEventListener('click', fn, el);
            el.addEventListener('click', e => eventbus.dispatchEvent(e, el));
        }
    });
}

Use in a component

<template>
  <button data-action="greet">Say Hello</button>
  <button data-action="celebrate">Celebrate</button>
  <output id="out"></output>
</template>

<script>
  const { bindActions } = await import('/src/bindActions.js');
  const out = shadowDocument.getElementById('out');

  function greet()     { out.value = 'Hello!'; }
  function celebrate() { out.value = 'Party!'; }

  bindActions(shadowDocument, { greet, celebrate });
</script>

08 — Best Practices

Best Practices

Follow the filename convention

/src/components/{prefix}/{prefix}-{name}.html

Example: page/page-home.html → <page-home>

Always pass host to the event bus

eventbus.addEventListener('my-event', handler, host);

Listeners are automatically cleaned up when the component disconnects.

Use dynamic import()

// Static import won't work inside component scripts
const { helper } = await import('/src/helper.js');

Don't nest <script> inside <template>

<!-- Wrong -->
<template>
  <div>...</div>
  <script>...</script>  <!-- ✗ -->
</template>

<!-- Correct -->
<template>
  <div>...</div>
</template>
<script>...</script>  <!-- ✓ -->

Don't use onclick in templates

<!-- Won't work -->
<button onclick="fn()">Click</button>

<!-- Use data-action -->
<button data-action="fn">Click</button>
