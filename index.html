Quick Start

Vanilla SPA is built for AI-ready development of W3C standards-conforming web-components. No magic, no virtual DOM — just the platform.

Installation

Grab the boilerplate code

git clone https://github.com/vanillaspa/boilerplate.git

or set up manually:

1

Create your project

npm init -y
npm install @vanillaspa/web-components @vanillaspa/event-bus @vanillaspa/sqlite-database
npm install vite --save-dev
2

Configure Scripts

{
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}

For SQLite WASM, you MUST configure Vite!

3

Set Up the Folder Structure

project/
├── index.html
├── package.json
└── src/
    ├── components/
    │   └── app/
    │        └── app-start.html
    └── main.js

@vanillaspa/web-components scans /src/components. All found components are added automagically to your customElements registry.

4

Run Dev Server

npm run dev

Your First Component

Filename Convention:
/src/components/{prefix}/{prefix}-{name}.html
Supports 2-fold through 4-fold names:
app-start.html → <app-start>
app-start-page-home.html → <app-start-page-home>

File: /src/components/app/app-start.html

<template>
  <header>
    <h1>My custom element</h1>
  </header>
  <main>
    <p>Hello World!</p>
  </main>
</template>

<style>
  :host { display: block; }

  header {
    padding: 1rem 2rem;
    background: #f9fafb;
    border-bottom: 1px solid #e5e7eb;
  }

  h1 {
    color: #ff69b4;
    font-size: 1.5rem;
  }
</style>

<script>
  // shadowDocument is automatically available!
  const h1 = shadowDocument.querySelector('h1');
  console.log('Component loaded:', h1.textContent);
</script>
How it works: The <script> body is compiled into an AsyncFunction with shadowDocument (your shadow root) passed as a parameter. Top-level await and dynamic import() work out of the box.
Important: <template>, <style>, and <script> must be siblings at the top level — not nested.

Web-Components

Vanilla SPA passes shadowDocument directly into every component script. It's your component's Shadow Root — no setup needed. 

<script>
  // Direct Shadow DOM access
  const button = shadowDocument.getElementById('my-btn');
  const output = shadowDocument.querySelector('.result');

  button.addEventListener('click', () => {
    output.textContent = 'Clicked!';
  });

  // Access the host element
  const host = shadowDocument.host;
  console.log(host.tagName); // "APP-START"
</script>
Note: Scripts run as async functions, not ES modules. Use dynamic import() instead of static import statements. Use the data-action pattern for functions in templates.

Event Bus

Use @vanillaspa/event-bus for context-scoped events. Listeners are automatically cleaned up when components disconnect.

Setup in src/main.js:

import('@vanillaspa/web-components');

customElements.whenDefined("app-start").then(() => {
    Promise.all([
        import('@vanillaspa/event-bus'),
        import('@vanillaspa/sqlite-database')
    ]).then((importedModules) => {
        importedModules.forEach((module) => {
            window[module.name] = Object.freeze({ ...module });
        });
    }).finally(() => {
        document.body.replaceChildren();
        document.body.appendChild(document.createElement('app-start'));
    });
})

In your component:

<script>
const host = shadowDocument.host;

// Listen for events
eventbus.addEventListener('user:action', (e) => {
    console.log('Action:', e.detail.action);
}, host);

// Fire an event
eventbus.dispatchEvent(
    new CustomEvent('user:action', {
        detail: { action: 'clicked' }
    }),
    host  // Context: only this instance's listeners
);
</script>

Event Contract

Use @vanillaspa/event-bus/contract for typed request/response communication between components via the 4-fold event naming convention.

Define a contract: src/contracts/sqlite-contract.js

import { defineContract } from '@vanillaspa/event-bus/contract';

export const SqliteContract = defineContract(window.eventbus, {
    namespace: 'sqlite',
    events: {
        query:     { detail: { sql: 'string', name: 'string' },  past: 'queried'  },
        statement: { detail: { sql: 'string', values: 'array',
                               name: 'string' },                 past: 'executed' },
    }
});

Bridge side (handles requests):

<script>
const bridge = SqliteContract.createBridge(shadowDocument.host);

bridge.handle('statement', async ({ sql, values, name }) => {
    return await sqlite.executeStatement(sql, values, name);
});
</script>

Consumer side (sends requests):

<script>
const consumer = SqliteContract.createConsumer(shadowDocument.host);

const rows = await consumer.request('query', {
    sql: 'SELECT * FROM notes', name: 'mydb'
});
console.log(rows);
</script>

Database

Add @vanillaspa/sqlite-database for a full SQLite database in the browser. Persisted in OPFS, runs in a dedicated Web Worker per database. 

<script>
// Create database
await sqlite.createDB('my-app');

// Create table
await sqlite.executeQuery(`
  CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT,
    email TEXT
  )
`, 'my-app');

// Insert data (parameterised — SQL injection safe)
await sqlite.executeStatement(
  'INSERT INTO users (name, email) VALUES (?, ?)',
  ['Anna', 'anna@example.com'],
  'my-app'
);

// Query
const users = await sqlite.executeQuery(
  'SELECT * FROM users',
  'my-app'
);
</script>

Using Functions in Templates

Since component scripts run as async functions, declared functions aren't global. The solution: data-action pattern + bindActions()

Create: /src/bindActions.js

export function bindActions(shadowDocument, actions) {
  shadowDocument.querySelectorAll('[data-action]').forEach(el => {
    const fn = actions[el.dataset.action];
    if (fn) {
      eventbus.addEventListener('click', fn, el);
      el.addEventListener('click', e => eventbus.dispatchEvent(e, el));
    }
  });
}

Use in component:

<template>
  <button data-action="greet">Say Hello</button>
  <button data-action="celebrate">Celebrate</button>
  <output id="out"></output>
</template>

<script>
  const { bindActions } = await import('/src/bindActions.js');

  const out = shadowDocument.getElementById('out');

  function greet() { out.value = 'Hello!'; }
  function celebrate() { out.value = 'Party!'; }

  bindActions(shadowDocument, { greet, celebrate });
</script>

Best Practices

Follow the filename convention

/src/components/{prefix}/{prefix}-{name}.html

Example: page/page-home.html → <page-home>

Use event-bus with context

eventbus.addEventListener('my-event', handler, host);

Automatic cleanup when component disconnects

Use dynamic import()

// Static import won't work inside component scripts
const { helper } = await import('./my-helper.js');

Don't nest script inside template

<!-- Wrong -->
<template>
  <div>...</div>
  <script>...</script>
</template>

<!-- Correct -->
<template>
  <div>...</div>
</template>
<script>...</script>

Don't use onclick in templates

<!-- Won't work -->
<button onclick="fn()">Click</button>

<!-- Use data-action -->
<button data-action="fn">Click</button>
